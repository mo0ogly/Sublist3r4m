# Security Policy

## Supported Versions

| Version | Supported          |
|---------|--------------------|
| 1.0.x   | Yes                |

## Reporting a Vulnerability

If you discover a security vulnerability in Sublist3r4m, please report it responsibly.

**Do NOT open a public GitHub issue for security vulnerabilities.**

Instead, please send a detailed report to the maintainer via the contact information in the repository profile, including:

1. Description of the vulnerability
2. Steps to reproduce
3. Potential impact
4. Suggested fix (if any)

You can expect an initial response within 72 hours.

## Scope

This policy covers:
- The Sublist3r4m codebase
- The subbrute module
- The JARVIS intelligence module
- The owner research engine

## Out of Scope

- Third-party APIs used by the tool (Shodan, VirusTotal, etc.)
- The original Sublist3r project (report upstream)
- Issues caused by user misconfiguration
